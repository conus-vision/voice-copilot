# Video Pitch

## Short version

Voice Copilot is a listening-first companion for coding agents. A separate commentator LLM
analyzes what Claude Code, Codex, or another CLI agent is doing in parallel,
summarizes the important reasoning and response, and lets you step in by voice
at the right moment.

It is not voice dictation for prompts, and it is not raw TTS for terminal logs.

Instead of babysitting terminal output, you listen, keep context, and intervene
only when needed.

## 45-60 second launch script

1. Hook

"Most tools for coding agents assume you are staring at the terminal the whole
time. Voice Copilot is for the opposite workflow."

2. What it is

"Voice Copilot runs a separate commentator LLM next to your coding agent. It
analyzes what the agent is doing, compresses the reasoning and response into
short spoken updates, and keeps you informed without constant reading."

3. User value

"You mainly listen. You can still read the trace when needed. And if the agent
goes in the wrong direction, you can interrupt and steer it at the right time."

4. Clarify what it is not

"It is not prompt dictation and it is not just reading raw LLM output aloud. It
is a companion layer that summarizes what matters."

5. Compatibility

"It works with coding CLIs like Claude Code and Codex, and it can proxy other
agent workflows too."

6. Close

"The goal is simple: less terminal babysitting, lower cognitive load, and a
better way to work with coding agents."

## Demo beats

1. Show the coding CLI already running.
2. Show Voice Copilot speaking a short summary.
3. Show that the trace exists, but is secondary to listening.
4. Interrupt by voice with a correction.
5. Show the agent change direction.
6. End on the product line: listening-first control for coding agents.
